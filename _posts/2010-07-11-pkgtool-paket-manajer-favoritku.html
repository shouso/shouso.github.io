Setelah hampir dua tahun menggunakan GNU/Linux berbagai jenis distro, akhirnya kutemukan juga paket manajer favoritku. Bukan dpkg atau apt. Bukan pula yum maupun pacman. Tapi aku telah jatuh hati pada pkgtool. Yah, memang pkgtool tidak punya GUI menawan, ibarat wanita, pkgtool punya wajah biasa saja, tanpa polesan, tetapi dibalik wajah bersahajanya itu, paket manajer utamanya Slackware Linux ini menyimpan berjuta pesona-kata sales sekarang inner beauty-mudah, sederhana, tapi berbahaya.


Mudah, pkgtool memang sangat mudah digunakan, dipindah-pindah antar distro maupun antar platform 32 bit dan 64 bit. Ga perlu repot mencari binari yang sesuai dengan platform CPU karena pkgtool hanya berupa shell script berjumlah 6 buah yakni :

1. pkgtool : shell script utama untuk masuk ke menu manajemen paket pkgtool.

2. installpkg : digunakan untuk instalasi program.

3. removepkg : digunakan untuk uninstall paket.

4. upgradepkg : digunakan untuk upgrade software.

5. makepkg : digunakan untuk membuat paket software untuk distro Slackware yang biasanya berakhiran tgz atau txz.

6. explodepkg : digunakan untuk ekstraksi paket software, tidak dilakukan secara default melalui script pkgtool.

Mudah, karena pkgtool hanya butuh di ekstrak dari kompresinya, lalu pindahkan hasil ekstraksi yang ada di folder sbin ke folder sbin sistem anda. Ga peduli anda pake versi yang 32 bit ataupun 64 bit, selama anda ekstrak paket dari file .tgz nya dipastikan bisa dipakai asalkan paket tar, xz, dan dialog terpasang di sistem.

Yah, hanya tar, xz, dan dialog yang secara khusus dibutuhkan  untuk menggunakan pkgtool. Paket tar digunakan untuk melakukan kompresi dan ekstraksi paket yang berakhiran .tgz dan pkgtool hanya stabil jika paket tar menyertakan versi 1.13 di dalam paketnya. Hanya ini bug yang ditemui selama menggunakan pkgtool.

xz dibutuhkan jika ingin dapat menggunakannya untuk instalasi paket-paket slackware versi 13.0 keatas. Karena, mulai Slackware 13.0, paket slackware menggunakan kompresi xz yang lebih efisien dan lebih cepat.  Untuk paket berekstensi .tgz, tentunya dapat di pasang menggunakan pkgtool dalam sistem GNU/Linux secara umum, karena paket gzip dapat dipastikan selalu ada di setiap sistem GNU/Linux.

dialog adalah paket program yang digunakan untuk menampilkan dialog -dialog di dalam terminal-bash misalnya- sehingga jika ingin melihat tampilan pkgtool, paket dialog ini harus ada. Untuk beberapa distro, paket ini sudah tersedia (saya coba di Arch, dan ArchBang dialog nya sudah ada), tetapi mungkin di distro lain paket ini tidak tersedia, contohnya Debian, Ubuntu, dan Ubuntu studio tidak menyertakannya dalam sistemnya. Maka, pilihan yang paling mudah adalah, pasang sendiri dialog menggunakan installpkg sesuai dengan platform sistem GNU/Linux anda, lalu nikmati tampilan pkgtool.

Sederhana, pkgtool adalah paket manajer yang paling sederhana, ga perlu update repository maupun melakukan pengecekan dependensi. asalkan ada paket yang sesuai (berekstensi tgz atau txz) maka proses instalasi program dapat dilakukan.

Sederhana, karena pkgtool tidak punya tampilan GUI yang menawan macam Synaptic, yang ada hanya dialog di terminal dan perintah yang hanya enam buah terasa sangat sederhana jika dibandingkan dengan aptitudenya debian yang mirip banget sama MC.

Berbahaya, memang dibalik cinta seringkali mengundang bahaya. Sering kali cinta membutakan mata dan membuat tuli telinga orang yang terserang oleh virus cinta. Begitu pula dengan pkgtool, dibalik pesonanya pkgtool menyimpan bahaya, jika tidak berhati-hati mengelolanya, konsekuensnya sistem anda bisa-bisa rusak dan tidak dapat digunakan. Penyebabnya adalah pkgtool tidak melakukan pengecekan dependensi sehingga kemungkinan tidak berjalannya paket akibat tidak/kurang dependensi besar. pengecekan dependensi dalam Slackware Linux sepenuhnya diserahkan kepada pribadi masing-masing admin sistem-sangat berbeda dengan keluarga Debian maupun Redhat. Sehingga, sebelum melakukan instalasi paket, admin harus memastikan bahwa paket yang dibutuhkan oleh program yang akan di pasang sudah tersedia.

Berbahaya, bahaya kedua dari pkgtool adalah pkgtool tidak melakukan pengecekan atas kemungkinan terjadinya double paket maupun kemungkinan satu paket menghapus paket yang sejenis dan dibutuhkan sistem. Jika terjadi, dan banyak paket penting yang terhapus, bisa dipastikan sistem anda akan ngadat dan tidak dapat dipakai.

Dan inilah yang terjadi pada Debian di PC ku, akibat kecerobohan dan cintaku pada pkgtool, Debian 5.04 yang terpasang di PC ber CPU amd Athlon64 dan berchip GPU Nvidia seri 6 milikku tidak dapat dipakai. sungguh pelajaran yang harus nya membuatku makin paham dengan bahaya dibalik kecantikan pkgtool, paket manajer favoritku.
